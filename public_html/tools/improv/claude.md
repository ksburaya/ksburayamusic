## Архитектура: матрица «лад × этап»

```
Лад (один из 10)
├── Знакомство (обязательно, новый слой)
│   ├── Этап 1: главная нота (тоника)
│   ├── Этап 2: 2 ноты
│   ├── Этап 3: 3 ноты
│   ├── Этап 4: 4 ноты
│   └── Этап 5: весь лад
└── Режимы практики (открываются после знакомства)
    ├── Короткий
    ├── Исследование
    └── Погружение
```

**Блокировки:**
- Знакомство обязательно для **каждого** лада
- Режимы практики для конкретного лада недоступны, пока знакомство этого лада не завершено (этап 5 не пройден)
- Этапы знакомства между собой не блокируются — кнопка «дальше» доступна всегда
- Знакомство с одним ладом не открывает практику для других ладов

**Точка входа:** карта 10 ладов с индикацией статуса каждого:
- не начат
- знакомство в процессе (виден этап, на котором остановился)
- знакомство завершено → доступны режимы практики

## Слой «Знакомство» — детально

### Цель

Услышать отношения ступеней к тонике. Это слуховая работа. **Никаких параметров выразительности** (динамика, темп, штрихи, длительности, регистр, движение рук) — они появляются только в режимах практики.

### Раскладка экрана

Наследует структуру эталонного скриншота, но без карточек параметров.

**Хедер :**
- Слева: `ТРЕНАЖЁР · ЗНАКОМСТВО · [название лада]`
- Справа: индикатор `ЭТАП N / 5`
- Тумблер режима отображения нот (см. ниже) — где-то в правой части хедера
- Кнопка «БЫСТРЫЙ» на знакомстве не нужна — скрыть

**Левая колонка:**
- `ЗВУКИ ЗАДАНИЯ` — текстом
- Интервальное отношение между нотами этапа (например, «малая терция») — только если нот ≥ 2
- Полный лад выписан мелким шрифтом, активные ноты этапа выделены: `СИ · ДО# · РЕ · МИ · ФА#` (СИ и РЕ выделены)

**Центральный модуль:**
- Большие ноты этапа (одна на этапе 1, две на этапе 2, и т.д.)
- В режиме кириллицы — крупно кириллица сверху, мелкая дублирующая латиница снизу: `Си·Ре` / `B·D`
- В режиме латиницы — крупно латиница, дублирующая строка снизу **отсутствует**

**Правая колонка:**
- `ИНСТРУКЦИЯ` — короткий текст к этапу («Услышь, как Си звучит сама по себе» / «Сопоставь два звука как разговор двух голосов»)

**Модуль фактурной подсказки (под центральным модулем):**
- Словесная подсказка (бытовая, без теории)
- Иконка-кнопка «обновить» рядом
- Подсказки меняются автоматически каждые **4 минуты** (значение настраиваемое, диапазон 3–5)
- Ручное «обновить» сбрасывает таймер автообновления
- Подсказка соответствует значению фактурного параметра (см. систематизацию ниже)

**Нижняя панель:**
- Время сессии — счёт времени вперёд, **не отсчёт**
- Кнопка `ПАУЗА`
- Кнопка `ДАЛЬШЕ` — переход на следующий этап, доступна всегда

### Логика этапов

**Этап 1 — главная нота:**
Тоника лада. Сразу экран с одной нотой и фактурной подсказкой. Ученик играет/слушает, жмёт «ДАЛЬШЕ».

**Этапы 2–4 — добавление нот:**
В начале этапа — экран выбора пути:
- **Слуховой путь (по умолчанию):** показывается список доступных ступеней лада, ученик выбирает следующую ноту на слух. Тренажёр не слушает звук с клавиатуры — ученик играет на своём инструменте, оценивает на слух, нажимает на выбранную ноту в интерфейсе и подтверждает «выбрал».
- **Случайный путь (тумблер):** генератор выдаёт случайную ноту из лада (не из уже выбранных).

После выбора нот — стандартный экран этапа.

**Этап 5 — весь лад:**
Все ступени активны. Выбор пути не нужен. Фактурная подсказка предлагает, как разыгрывать полный материал.

### Завершение знакомства

После прохождения этапа 5 — экран «знакомство пройдено». Открываются 3 режима практики для этого лада.

## Систематизация фактур

### Параметры

**Ось 1: Диапазон**
- `limited` — внутри октавы / одного регистра
- `full` — любые регистры

**Ось 2: Тип игры одной рукой**
- `chord_simultaneous` — аккорды одновременные
- `chord_split` — аккорды разделённые (1+3, 2+2)
- `cluster` — кластеры
- `arpeggio_up` / `arpeggio_down` / `arpeggio_mixed` — арпеджио
- `continuous_motion` — постоянное движение (соседние ноты со смещением)
- `ripples` — переливы
- `melody` — мелодия (одна линия)

**Ось 3: Координация рук**
- `one_hand` — одна рука играет, вторая молчит/держит
- `both_same` — обе руки одинаковый тип
- `both_different` — обе руки разные типы

### Ограничения по количеству нот

| Кол-во нот | Доступные значения Оси 2 |
|---|---|
| 1 | `melody` (повтор той же ноты), `chord_simultaneous` (та же нота в разных октавах) |
| 2 | + `arpeggio_*` (из 2 нот), `chord_split` простые |
| 3 | + `chord_simultaneous` (трезвучие), `ripples` (простые), `arpeggio_*` расширяется |
| 4+ | вся палитра |

### Связь с UI

В коде фактура — значение параметра.
В UI — **массив словесных формулировок** на каждое значение.

Кнопка «обновить» / автообновление берёт следующую формулировку из массива текущего значения. Когда массив заканчивается — переключается на другое значение параметра, подходящее для текущего количества нот.

**Принцип формулировок:** бытовые, без теоретических терминов («арпеджио», «триоль» и т.п.), приглашающая интонация, короткие. Автору нужна возможность редактировать эти формулировки — вынести в отдельный файл данных (JSON или подобное).

**Пример** для `arpeggio_up`:
```json
[
  "перебирай ноты снизу вверх неторопливо",
  "попробуй пройтись по этим звукам как по ступенькам вверх",
  "вверх — по одной ноте за раз, спокойно"
]
```

Изначальный наполненный набор формулировок собрать самостоятельно (минимум 3 на каждое значение), отметить файл как «требует ревизии автором».

## Переключатель режима отображения нот

**Два режима:**
- Кириллица — `До · Ре · Ми · Фа · Соль · Ля · Си` (по умолчанию)
- Латиница — `C · D · E · F · G · A · B`

**Переключатель:**
- Глобальная настройка
- В хедере справа, рядом с индикаторами уровня/этапа
- Сохраняется между сессиями

**Применяется к:**
- Названию лада в хедере (`Си — эолийский` / `B — aeolian`)
- Звукам задания в левой колонке
- Большим нотам в центральном модуле
- Списку ступеней лада на экране выбора пути

**Дублирующая мелкая строка** под центральным модулем (на эталоне `SI · RE` под `Си · Ре`):
- Видна **только в режиме кириллицы**
- В режиме латиницы скрывается

**Диезы/бемоли:**
- Кириллица: `Сиb`, `Ре#`
- Латиница: `Bb`, `D#`
- В каждом ладе использовать единообразно — все ноты через бемоли или все через бемоли, в зависимости от того, как лад принято записывать (например, ре-минор → бемоли)

## Сохранение прогресса

`localStorage`, формат:

```json
{
  "settings": {
    "note_notation": "cyrillic",
    "hint_refresh_interval_minutes": 4
  },
  "lads": {
    "aeolian_B": {
      "introduction": {
        "completed": false,
        "current_stage": 3,
        "notes_chosen_per_stage": {
          "2": ["B", "D"],
          "3": ["B", "D", "F#"]
        }
      },
      "last_practice_mode": null
    },
    "ionian_C": {
      "introduction": {
        "completed": true,
        "current_stage": 5
      },
      "last_practice_mode": "exploration"
    }
  }
}
```

При возврате на карту ладов — состояние читается из `localStorage`.
