# Delay Tool — улучшения для учеников без опыта

## Контекст

Педагогический инструмент для знакомства с live-looping. Ученик подключает инструмент через USB-аудиоинтерфейс к ноутбуку. Проблема: после подключения интерфейса звук не идёт в колонки ноутбука — нет возможности выбрать устройства ввода и вывода.

## Цель

Добавить разовую настройку устройств перед подключением и кнопку сброса буфера.

## Изменения интерфейса

### Порядок элементов (сверху вниз)

1. Заголовок `Delay` (без изменений)
2. Дропдаун **Вход** — список `audioinput`-устройств системы
3. Дропдаун **Выход** — список `audiooutput`-устройств системы
4. Кнопка **Подключить вход** / **Отключить** (без изменений)
5. Слайдеры Время / Затухание / Баланс (без изменений)
6. Визуализатор (без изменений)
7. Кнопка **Сброс** — только при активном подключении

### Стиль

Тёмный, моноширинный — без изменений. Дропдауны в том же стиле что кнопка «Подключить»: transparent background, border `#3a3a3a`, font Courier New.

## Поведение

### Инициализация устройств

При загрузке страницы:
1. Короткий `getUserMedia({ audio: true })` — нужен чтобы браузер раскрыл реальные имена устройств в `enumerateDevices()`
2. Немедленно остановить полученный поток (`stream.getTracks().forEach(t => t.stop())`)
3. `navigator.mediaDevices.enumerateDevices()` → отфильтровать `audioinput` и `audiooutput`
4. Заполнить дропдауны; если устройство одно — выбрать автоматически

### Подключение

- При нажатии «Подключить»: `getUserMedia` с `deviceId: { exact: selectedInputId }`
- После создания `AudioContext`: `ctx.setSinkId(selectedOutputId)` (Chrome 110+)
- Дропдауны блокируются (`disabled`), разблокируются только после «Отключить»

### Сброс буфера

Кнопка **Сброс** активна только при `running === true`.

При нажатии:
1. Отключить все узлы текущего AudioContext
2. Закрыть AudioContext (`ctx.close()`)
3. Создать новый AudioContext с тем же MediaStream
4. Пересоздать граф узлов с текущими значениями слайдеров
5. Применить `setSinkId` к новому контексту

Результат: delay-буфер обнулён, слайдеры и устройства не меняются.

## Технические ограничения

- `setSinkId()` доступен в Chrome 110+; инструмент уже ограничен Chrome по CLAUDE.md
- Если `setSinkId` не поддерживается (`typeof ctx.setSinkId !== 'function'`): скрыть дропдаун «Выход», показать текст «Выход управляется системой»
- `enumerateDevices()` без предварительного `getUserMedia` возвращает устройства с пустыми label — поэтому нужен предварительный разрешительный запрос

## Что не меняется

- Аудиограф (DelayNode → GainNode feedback loop → dry/wet mix)
- Ограничение feedback не выше 0.95
- `setTargetAtTime()` при изменении времени
- `echoCancellation: false, noiseSuppression: false, autoGainControl: false`
- Визуализатор
